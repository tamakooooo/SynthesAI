核心要点格式

1. 专为NAS用户设计的哔哩哔哩视频同步下载工具
2. 基于Rust+Tokio构建，支持异步并发与分块并行下载

  • 缺少标题-内容分离，难以布局

新格式（v2.0）

核心要点格式

专为 NAS 用户 ：针对网络附属存储场景优化，满足私人影音库需求
Rust + Tokio 架构 ：高性能异步并发，资源占用低，稳定性强
智能选流下载 ：自动选择最佳画质，FFmpeg 自动合并音视频流
多源支持 ：收藏夹、稍后再看、UP主投稿、视频合集全覆盖
媒体库兼容 ：采用 Jellyfin/Emby 兼容命名，无缝导入媒体服务器
优势：
  • 关键词简洁（3-8字），适合作为卡片标题
  • 描述详细（15-30字），提供具体信息
  • 4-6个要点，完美适配网格布局
  • 标题-内容分离，层次清晰

编辑风格知识卡片完整布局

新格式的数据将完美适配以下卡片区域：

框架网格卡片

来源：key_points 前 4-6 个

  • 每个要点生成一张卡片
  • 徽章编号：1-6
  • 卡片标题：关键词部分
  • 卡片描述：详细描述部分

深色面板（左侧）

内容摘要 + 关键概念

  • 内容摘要：summary 字段
  • 关键概念：key_concepts
  • 概念名称：term
  • 定义解释：definition

浅色面板（右侧）

核心洞察（带编号）

  • 来源：key_points 前 3 个
  • 编号：1-3（Playfair Display）
  • 标题：关键词部分
  • 描述：详细描述部分

底部高亮条

关键词标签

  • 来源：tags 字段
  • 显示：带 # 符号
  • 格式：关键词 · #标签1 · #标签2 · #标签3

提示词模板 JSON Schema

新增格式验证

key_points:
  type: array
  items:
    type: string
    pattern: "^.{3,8}[：:].{15,30}$"  # 正则表达式验证
  minItems: 4
  maxItems: 6
  description: 核心要点列表（格式：关键词：详细描述）
验证说明：
  • .{3,8} - 关关键词 3-8 个字符
  • [：:] - 支持中文冒号或英文冒号
  • .{15,30} - 详细描述 15-30 个字符
  • 确保每个要点都符合标准格式

总结

新的提示词格式（v2.0）完美适配编辑风格知识卡片布局，通过"关键词：详细描述"的格式，实现了：

  • ✅ 标题-内容分离，层次清晰
  • ✅ 适合网格卡片布局展示
  • ✅ 便于快速阅读和理解
  • ✅ 提升知识卡片的专业性
